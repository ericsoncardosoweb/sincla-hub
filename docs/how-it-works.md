## 4. Texto Detalhado de Cada Passo

### Passo 1: Cadastre voce

**Numero:** 1
**Icone:** User com cartao de identidade / ID Badge
**Titulo:** Cadastre voce
**Corpo:**
```
Seus dados pessoais — nome, e-mail, telefone —
viram sua identidade unica no ecossistema Sincla.
Um login para tudo. Uma senha para sempre.
```

**Destaque visual:** Avatar/silhueta com badge de verificado

**Racional:**
- Começa pelo indivíduo (mais pessoal)
- "Identidade única" reforça o diferencial
- "Um login para tudo" resolve a dor de múltiplas senhas

---

### Passo 2: Cadastre sua empresa

**Numero:** 2
**Icone:** Prédio / Building com documento
**Titulo:** Cadastre sua empresa
**Corpo:**
```
Digite o CNPJ e buscamos os dados automaticamente.
Nome fantasia, segmento, tamanho — tudo registrado uma vez
e compartilhado em todos os modulos que voce ativar.
```

**Destaque visual:** Campo de CNPJ com auto-complete visual

**Racional:**
- Mostra que é fácil (busca automática)
- "Compartilhado em todos os módulos" é o diferencial chave
- Menciona dados específicos para tangibilizar

---

### Passo 3: Ative modulos

**Numero:** 3
**Icone:** Grid de apps / Módulos interconectados
**Titulo:** Ative modulos
**Corpo:**
```
Escolha RH, EAD, Financas, Leads ou Agenda.
Ao abrir qualquer modulo, seus dados pessoais
e da empresa ja estarao preenchidos.
Adicione mais modulos quando quiser — zero retrabalho.
```

**Destaque visual:** Cards de módulos com checkmarks

**Racional:**
- Lista módulos específicos (tangível)
- "Já estarão preenchidos" é a promessa central
- "Zero retrabalho" fecha com o benefício

## 5. Elemento Visual Central: Diagrama de Fluxo

### Diagrama de Dados

Abaixo dos 3 passos, mostrar visualmente como os dados fluem:

```
+------------------+
|    VOCE          |
|  Nome, Email,    |
|  Telefone        |
+--------+---------+
         |
         v
+------------------+
|  SUA EMPRESA     |
|  CNPJ, Nome,     |
|  Segmento        |
+--------+---------+
         |
    +----+----+----+----+
    |    |    |    |    |
    v    v    v    v    v
  +--+ +--+ +--+ +--+ +--+
  |RH| |ED| |BO| |LE| |AG|
  +--+ +--+ +--+ +--+ +--+

  Todos com seus dados prontos.
```

**Animacao sugerida:**
1. Dados pessoais aparecem (fade-in)
2. Dados da empresa aparecem abaixo (fade-in com delay)
3. Linhas conectam aos módulos (desenho progressivo)
4. Módulos "acendem" em sequência

## 10. Tom de Voz

### O Que Fazer
- Ser direto: "Cadastre uma vez" > "Simplifique seu processo de cadastro"
- Usar números: "1 cadastro", "zero retrabalho"
- Mostrar benefício: "Seus dados já estarão lá"
- Gerar identificação: "Cansado de preencher os mesmos dados?"

### O Que Evitar
- Jargão técnico: "SSO", "sincronização de dados", "API"
- Superlativos: "a melhor solução", "revolucionário"
- Vagueza: "experiência integrada", "plataforma unificada"

## 12. Animacoes

### Entrada da Secao
- Badge: fade-in
- Título: fade-in-up (100ms delay)
- Subtítulo: fade-in-up (200ms delay)

### Passos
- Stagger de 150ms entre cada passo
- Cada passo: fade-in-up

### Diagrama
- Trigger: quando entra 50% no viewport
- Animação de "dados descendo" dos boxes para os módulos
- Duração total: 2 segundos
- Deve parecer "dados fluindo"

### Comparativo
- Fade-in simples
- Checkmarks podem ter micro-animação de "aparecer"
